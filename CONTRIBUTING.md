First off, thank you for considering contributing to thedecoder! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## Code of Conduct / 행동 강령
We follow a professional and respectful interaction policy. Please be kind to other contributors.
우리는 전문적이고 존중하는 상호작용 정책을 따릅니다. 다른 기여지분들께 친절하게 대해 주십시오.

## How to Contribute / 기여 방법

### 1. Reporting Bugs / 버그 보고
If you find a bug, please open an issue on GitHub with a clear description and steps to reproduce.
버그를 발견하면 명확한 설명과 재현 단계를 포함하여 GitHub에 이슈를 열어 주십시오.

### 2. Feature Requests / 기능 제안
We always look for ways to improve! Use GitHub Issues to suggest new "Monster Grade" features.
우리는 항상 개선 방법을 찾고 있습니다! 새로운 "Monster Grade" 기능을 제안하려면 GitHub 이슈를 사용하십시오.

### 3. Pull Requests / 풀 리퀘스트
- Fork the repository. / 리포지토리를 포크하십시오.
- Create a new branch for your feature or fix. / 기능 또는 수정을 위한 새로운 브랜치를 생성하십시오.
- Ensure your code follows the premium aesthetics and clean code standards. / 코드가 프리미엄 미학과 클린 코드 표준을 따르는지 확인하십시오.
- All code must include bilingual documentation where applicable. / 모든 코드는 가능한 경우 한영병기 문서화를 포함해야 합니다.
- Submit a PR with a concise description of your changes. / 변경 사항에 대한 간결한 설명과 함께 PR을 제출하십시오.

## Coding Standards / 코딩 표준
- **C++17**: Use modern C++ features. / 현대적인 C++ 기능을 사용하십시오.
- **Premium UI**: Follow the Samsung Notes inspired dark mode theme for any GUI changes. / GUI 변경 시 삼성 노트 스타일의 다크 모드 테마를 따르십시오.
- **Bilingualism**: Maintain the bilingual (Korean/English) documentation policy. / 한영병기 문서화 정책을 유지하십시오.
